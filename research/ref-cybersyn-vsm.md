# 参考文档：Project Cybersyn 与 生存系统模型 (VSM)

_这是为 chaoting-agent 架构设计提供的基础理论支撑库。_

## 1. 什么是 Project Cybersyn？
**Project Cybersyn（赛博协同工程）** 是 1971 年至 1973 年间智利政府（萨尔瓦多·阿连德总统时期）进行的一项极具前瞻性的实验。其目标是利用当时有限的计算机网络（主要是电传打字机 Telex 组成的网络）和“控制论 (Cybernetics)”原理，对全国国有化经济进行**实时的、去中心化的宏观调控与决策**。

虽然因为 1973 年的政变而夭折，但它那充满科幻感的“中央控制室 (Opsroom)”和背后的系统论思想，至今仍被视为**赛博国家的雏形**和**极简高层次决策系统**的鼻祖。

## 2. 核心理论支撑：生存系统模型 (Viable System Model, VSM)
Project Cybersyn 的架构由英国运筹学与控制论专家斯塔福德·比尔 (Stafford Beer) 设计，其核心是 **Viable System Model (VSM)**。

VSM 理论认为，任何能够在复杂多变的环境中生存下来（Viable）的组织或系统，都必须具备 5 个互相连接的功能子系统。这五个系统具有**递归性（分形结构）**，从整个国家到单个企业，再到单个人，都符合这一五层架构。

### VSM 的五个子系统 (System 1-5)

*   **System 1 (Operations / 执行与运营)**
    *   **定义**：直接执行系统主要职能的基本单元（如 Cybersyn 中的各个工厂）。它们在很大程度上是自治的，直接与外部环境互动。
*   **System 2 (Coordination / 协调与防震荡)**
    *   **定义**：防止 System 1 的各个自治单元之间发生冲突或“震荡”。它不发号施令，只提供共享信息和调度规则（如排班表、公用资源路由）。
*   **System 3 (Control / 控制与稳态)**
    *   **定义**：管理“此时此地 (Here and Now)”。负责监督 System 1 的运行，分配资源，确保内部的整体稳定和合规。它通常还包含一个 **System 3*** (审计通道)，用于直接、不定期的抽查。
*   **System 4 (Intelligence / 战略与规划)**
    *   **定义**：关注“彼时彼地 (There and Then)”。它负责向外看（外部环境）和向前看（未来趋势）。进行建模、推演、生成应对战略和规划方案。
*   **System 5 (Policy / 主权与身份)**
    *   **定义**：系统的最高裁决者。它定义系统的终极目标、价值观和边界。它平衡 System 3 (当前的稳定) 和 System 4 (未来的发展) 之间的矛盾，做出最终的策略定夺。

## 3. VSM 对我们 AI Agent 架构的启发

在我们的 AI 决策引擎中，完全抛弃了传统的“部门”隐喻，直接将 VSM 映射为代码架构：

1.  **用户** = System 5 (主权核，输入指令并进行最终准奏)
2.  **Planner Agent** = System 4 (神谕核，负责推演、规划蓝图)
3.  **Auditor Agent** = System 3 (神盾核，负责逻辑校验、风控，持否决权)
4.  **Router/Dispatcher Agent** = System 2 (枢纽核，负责协调和下发)
5.  **Execution Agents** = System 1 (具体的搜索/代码执行器)

这就是极其硬核的控制论映射，不带任何陈旧的拟人化官僚包袱。
